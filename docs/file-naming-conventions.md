# File Naming Conventions

## General Rule
ใช้ lowercase kebab-case สำหรับไฟล์เอกสารและคอนฟิก

## Source Files
- ใช้ชื่อที่บอกหน้าที่ชัดเจน เช่น todo-service.ts, todo-types.ts
- หลีกเลี่ยงชื่อกว้างเกินไป เช่น utils.ts

## Test Files
- เก็บใน tests/public/
- ตั้งชื่อให้บอกสิ่งที่ทดสอบ เช่น todo-service.test.ts

## Script Files
- เก็บใน scripts/
- ตั้งชื่อสะท้อนงาน เช่น check-student-rules.ts

## Documentation Files
- เก็บใน docs/
- ใช้ชื่อเช่น assignment-requirements.md, coding-conventions.md
